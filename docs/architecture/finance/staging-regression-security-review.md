## 1. Purpose and scope

Phase 3-12 is the **consolidated regression + security review** before Phase 3-13 (evidence pack update) and Phase 3-14 (Phase 4 go/no-go decision). It is a checkpoint, not a fresh implementation. Two questions:

1. **Regression:** does every code path that Slice 2 + Phase 3 touched still behave per its contract? (Answer: yes locally, per §4. Operator-side staging behavior is documented in the Phase 3-5 / 3-7 / 3-8 / 3-9 / 3-10 runbooks; those are dry-run-by-default and have empty evidence packs awaiting separately-authorized operator execution.)
2. **Security:** are the safety boundaries that make the staging activation arc honest still intact? (Answer: yes for every implementation-side boundary that can be verified from the committed code; live-staging security verification — PostgREST exclusion against the deployed instance, RLS catalog inventory, etc. — remains a Phase 3-3 + Phase 3-11 operator deliverable that is not duplicated here.)

3-12 is **not** a re-execution of every earlier Phase 3 packet. Each prior packet has its own evidence pack table waiting to be populated by the deploy owner. 3-12 consolidates:

- The local-verification evidence that the committed branch is consistent with itself (the §4 regression matrix's "executed locally" column).
- The implementation-side security boundaries that derive from code (the §5 security matrix's "implemented / verified by code" rows).

3-12 explicitly defers:

- Any live staging regression against the deployed VPS-1 workers + `staging-backend-heavy` route + staging Supabase.
- Any live security scan against the deployed instances (PostgREST surface check, RLS catalog inventory in the live DB, JWT inspection, network-level audit).
- The operator-execution sign-offs for Phase 3-7 / 3-8 / 3-9 / 3-10 — those execution evidence packs remain the responsibility of their respective runbooks.

**This document is the review packet.** No live action was taken by this task. Executing the operator-only checks in §4 + §5 is a separately authorized action covered by §6 + §7.
