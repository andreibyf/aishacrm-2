# Finance Ops ÔÇö Beta Limitations (Honest Deferrals)

**Date:** 2026-05-31
**Branch:** `feat/finance-ops-beta-implementation`
**Scope:** what the Finance Ops beta does **not** do, with the reason and the source signal that enforces the
deferral. Beta readiness here = accurate, consistent, immutable, accountable **recordkeeping** ÔÇö not external
accounting-system integration. These limitations are acceptable for beta per the packet's "Acceptable beta
limitations" list; they must not be presented to users as if implemented.

> **Beta-scope decision (2026-06-05):** Controlled COA wiring is **not** a read-only-beta blocker ÔÇö it is a **production-readiness prerequisite and a fast-follow**, gated alongside persistent-event activation and provider COA mapping (which it is also a prerequisite for). Rationale: the beta bar is *read-only recordkeeping integrity, not external accounting integration*, with durable storage, period-close, and provider sync all already deferred; an emergent COA is consistent with that posture **provided it is disclosed** (limitation 9 above) and provided statements stay correct (they do ÔÇö derivation is `classification`-based). Wiring lands via PR #643's Slice 1 before persistent/production activation. (An external review flagged COA as a potential "beta blocker" without this scope context; this note records the deliberate decision.)
