# Finance Ops — Beta Limitations (Honest Deferrals)

**Date:** 2026-05-31
**Branch:** `feat/finance-ops-beta-implementation`
**Scope:** what the Finance Ops beta does **not** do, with the reason and the source signal that enforces the
deferral. Beta readiness here = accurate, consistent, immutable, accountable **recordkeeping** — not external
accounting-system integration. These limitations are acceptable for beta per the packet's "Acceptable beta
limitations" list; they must not be presented to users as if implemented.

| #   | Limitation                                                                                                                                                        | Why deferred                                                                        | Enforced / signalled at                                                                                                                                                        |
| --- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 1   | **In-memory persistence** — finance state is not durably stored across process restarts                                                                           | Persistent event store + projection reads are Phase 4-1, gated behind staging proof | `data.source.mode = 'in_memory'` on every read (`backend/routes/finance.v2.js` `buildSource`)                                                                                  |
| 2   | **`runtime.mode` is a placeholder** (`mock_read_only`) — not an authoritative running-mode signal                                                                 | The domain service does not yet publish an authoritative mode value                 | gap `FINANCE_API_GAPS.runtimeMode` (§8.2.9) in `src/api/finance.js`; surfaced as an annotation, not authoritative                                                              |
| 3   | **Projection cursors / lag endpoint absent** (`GET /projection/cursors`)                                                                                          | Depends on the persistent projection store (Phase 4-1)                              | gap `FINANCE_API_GAPS.projectionCursors` (§8.2.6); UI renders a `GapStateCard`, not fake data                                                                                  |
| 4   | **Persistent-events route lift not enabled**                                                                                                                      | Fail-closed until implementation + tests + staging proof land                       | `ENABLE_FINANCE_PERSISTENT_EVENTS` fail-closed guard (`backend/routes/finance.v2.js:48`)                                                                                       |
| 5   | **Provider writes disabled** (ERPNext/QuickBooks/Xero/NetSuite)                                                                                                   | Beta bar is recordkeeping integrity, not external sync; writes are post-activation  | `FINANCE_PROVIDER_WRITES_ENABLED` default-closed (`backend/lib/finance/adapterJobProcessor.js:332-345`); `/adapters` is declarative metadata only                              |
| 6   | **Durable immutability storage** (append-only)                                                                                                                    | Pairs with persistent store (limitation 1/4)                                        | reversal-based correction works today (`accountingEngine.createReversalDraft`); durable append-only store deferred                                                             |
| 7   | **CSV/PDF export / recordkeeping download**                                                                                                                       | Sequenced after number-correctness (this slice). "Reports come afterwards."         | not present in `src/api/finance.js` (GET reads only); next slice                                                                                                               |
| 8   | **Balance-sheet "balanced" reflects a simplified engine** — revenue/expense are not auto-closed to equity, so a period with net income reads `is_balanced: false` | The engine has no period-close step; this is honest, not a bug                      | `accountingEngine.buildBalanceSheet` `is_balanced = assets == liabilities + equity`; UI shows an explicit "unbalanced (ledger integrity issue)" warning rather than masking it |

## Activation posture (unchanged by this slice)

No production action, no staging/Coolify/Doppler mutation, no migration application, no provider writes, no
env-flag flips. All 16 Phase 3-13 §7 guardrails preserved. This slice adds tests, one display-state change in
`LedgerSummary`, and documentation only.
