Repository Summary

AiSHA CRM

AI-native CRM and executive-assistant workflow platform, summarized from checked-in repo evidence only.

What It Is

AiSHA CRM is a React + Node.js customer relationship management platform positioned in the repo as an AI-powered Executive Assistant rather than a traditional CRM with AI bolted on.

It combines core CRM data, reporting, workflows, and AI tool execution through the custom Braid DSL and a multi-provider LLM layer.

Who It's For

Primary persona: sales leaders, operators, and CRM users who need one system to manage contacts, pipeline, activities, reporting, and AI-assisted work.

What It Does

  • Manages BizDev sources, leads, contacts, accounts, and opportunities across the CRM lifecycle.
  • Supports AI chat, executive-assistant workflows, and natural-language actions through backend AI routes and Braid tools.
  • Tracks activities, tasks, meetings, and notes tied to CRM records.
  • Runs dashboard, reporting, and analytics views, including pipeline and performance reporting.
  • Provides workflow automation, webhook execution, and queued workflow processing.
  • Includes AI campaigns, telephony integrations, and outbound call/email workflow support.

How It Works

  • Frontend: React 18 + Vite app in src/; browser code resolves VITE_AISHACRM_BACKEND_URL and calls backend REST endpoints.
  • Backend: Express server in backend/ mounts domain routes for CRM, AI, reports, workflows, auth, metrics, and system operations.
  • Data: Supabase/Postgres is the primary system of record with tenant UUID isolation and RLS; Redis memory/cache services support runtime coordination and caching.
  • AI/Automation: Braid tool definitions live in braid-llm-kit/examples/assistant/; backend AI uses a multi-provider engine and can call the separate Braid MCP server.
  • Jobs: backend workflow execution uses Bull; the distributed Braid MCP server uses BullMQ.
Browser UI -> Express API -> Supabase/Postgres
Parallel services: Redis memory/cache, Braid tools, AI engine, workflow queues, optional MCP server and n8n profile.

How To Run

  • Copy environment defaults: .env.example to .env; local Docker also reads .env.local.
  • Recommended start: run docker compose up -d --build.
  • Open the app at http://localhost:4000; backend API is at http://localhost:4001.
  • Minimal local dev alternative: npm install, npm run dev, then in backend/ run npm install and npm run dev.
